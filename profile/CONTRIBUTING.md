## How to Contribute

### 1. Reporting Issues
- Use GitHub Issues for bug reports and feature requests
- Search existing issues before creating new ones
- Provide clear descriptions and reproduction steps

### 2. Documentation
- Fix typos, improve clarity, add examples
- Translate content to Sinhala/Tamil
- Create tutorials and case studies

### 3. Code Contributions
- Fork the repository
- Create a feature branch: `git checkout -b feature/amazing-feature`
- Commit changes: `git commit -m 'Add amazing feature'`
- Push to branch: `git push origin feature/amazing-feature`
- Open a Pull Request

### 4. Content Guidelines
- Use HTML5 semantic markup
- Follow Tailwind CSS conventions
- Write accessible, mobile-responsive code
- Include comments for complex logic
- Test across browsers (Chrome, Firefox, Safari, Edge)

## Code Style

- **HTML**: Use 2-space indentation, semantic tags
- **CSS**: Tailwind utility classes, custom CSS in `assets/css/main.css`
- **JavaScript**: ES6+, vanilla JS only (no frameworks)
- **Markdown**: Follow standard Markdown syntax

## Review Process

1. Submit PR with clear description
2. Maintainers review within 7 days
3. Address feedback if requested
4. Approval and merge

## Community Standards

- Be respectful and inclusive
- Follow our [Code of Conduct](CODE_OF_CONDUCT.md)
- Help others learn and grow
